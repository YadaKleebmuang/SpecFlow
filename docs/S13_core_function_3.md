## 1. ขั้นตอนการทำงาน (Workflow)

ฟังก์ชันการให้บริการตอบคำถามเชิงความรู้ด้านฮาร์ดแวร์ไอที (Hardware FAQ) และการให้คำแนะนำปรับจูนซอฟต์แวร์เครื่อง (Software Tuning) ทำงานโดยไม่ต้องผ่านระบบเก็บรวบรวมฟอร์ม (Form-less execution) แต่จะประมวลผลและตอบกลับผ่านกฎแบบตรงตัว (Rule-based Direct Execution) ดังโฟลว์ขั้นตอนการทำงานต่อไปนี้:

```text
[ผู้ใช้] พิมพ์ถาม "การ์ดจอทำหน้าที่อะไร" หรือกดปุ่มสอบถามข้อมูล
  -> [Rasa NLU] ระบุเจตนา ask_gpu_info (หรือ ask_cpu_info, ask_ram_info, ask_ssd_hdd_diff, optimize_performance)
  -> [Rasa Core] จับคู่เข้ากฎ Rules ใน rules.yml เช่น "Answer FAQ GPU"
  -> [Rasa Core] เรียกใช้งาน Action Server พอร์ต 5055 ทันที: action_faq (หรือ action_optimize_performance)
  -> [Action Server] ตรวจ Intent ที่เข้ามา ดำเนินตรรกะสุ่มประโยคคำอธิบาย (Text Randomizer)
  -> [Action Server] โหลดแม่แบบ Flex Message JSON ที่สัมพันธ์กับหัวข้อ (เช่น card_faq_gpu.json) 
  -> [Action Server] เลือกข้อความดึงรูปภาพ และผสานโครงสร้าง Payload
  -> [Action Server] เรียกใช้งาน LineOutput push_message ส่งการ์ดสารสนเทศคืนห้องแชท LINE
  -> [ผู้ใช้] ได้รับการ์ดสรุปความรู้อ่านง่าย พร้อมปุ่มกดด่วนสำหรับเลือกถามหัวข้ออื่นต่อ
```

## 2. ตรรกะการประมวลผลทางธุรกิจ (Business Logic)

ตรรกะทางธุรกิจของระบบถามตอบประกอบด้วย 2 ส่วนย่อยหลักที่เขียนควบคุมในไฟล์ [actions.py](file:///c:/Users/thirs/Downloads/SpecFlow/app/rasa/actions/actions.py):

### 2.1. ตรรกะการสุ่มหมุนเวียนเนื้อหาการตอบกลับ (Text Variation Randomizer)
เพื่อหลีกเลี่ยงความจำเจและสร้างความเป็นธรรมชาติในฐานะระบบสนทนาอัตโนมัติ (Conversational Chatbot) ตัวประมวลผล `ActionFAQ` ได้รับการพัฒนาให้มีข้อมูลชุดข้อความอธิบายหัวข้อละ **3 รูปแบบ (Variations)** ซึ่งเน้นเนื้อหาเทคนิคที่ลึกต่างกัน:

* **ตรรกะการสุ่ม:**
  1. ดึงชุดข้อความอธิบายตามเจตนา (Intent) ที่สกัดได้
  2. รันชุดคำสั่งสร้างดัชนีแบบสุ่ม: `selected_idx = random.randint(0, len(texts) - 1)`
  3. นำดัชนีไปดึงอาร์เรย์ข้อความดั้งเดิมเพื่อนำส่งคืน (Fallback)
  4. ทำการดึงรายการ Flex Message Card จากไฟล์เทมเพลต และหยิบดัชนีกล่องการ์ดที่สัมพันธ์กัน (`card_payload[selected_idx]`) เพื่อส่งเป็น Payload UI บน LINE

---

### 2.2. ตรรกะแนะนำการปรับปรุงซอฟต์แวร์เครื่อง (Software Tuning Logic)
สำหรับเจตนา `optimize_performance` (เช่น ผู้พิมพ์ถามว่า "คอมช้ามากแก้ยังไง") ตัวประมวลผล `ActionOptimizePerformance` จะนำเสนอแนวทางการจูนระบบปฏิบัติการผ่านซอฟต์แวร์ 4 ประการ:

1. **การอัปเดตไดรเวอร์ (Driver Updates):** เน้นย้ำให้ผู้ใช้อัปเกรดไดรเวอร์การ์ดจอแสดงผลให้เป็นรุ่นล่าสุดเสมอ (NVIDIA GeForce/AMD Adrenalin)
2. **การปิดโปรแกรมทำงานเริ่มต้น (Disable Startup Programs):** แนะนำวิธีลดหน่วยความจำสูญเปล่าโดยการเปิด Task Manager ไปปิดโปรแกรมที่รันเองขณะบูตวินโดวส์
3. **การเคลียร์พื้นที่ไฟล์ขยะระบบ (Disk Cleanup):** ตรวจเช็คพื้นที่เก็บข้อมูลระบบไดรฟ์ C: ไม่ให้อัดแน่นจนระบบเขียน Page File ล่าช้า
4. **การตั้งค่าแผนพลังงานประสิทธิภาพสูง (High Performance Power Plan):** ปรับพลังงานใน Windows เพื่อรีดสัญญาณประมวลผลสูงสุด

## 3. หน้าจอและอินเทอร์เฟซโต้ตอบ (UI Screenshots & Layouts)

ฟังก์ชันถามตอบความรู้ใช้งานแม่แบบ Flex Message JSON จากโฟลเดอร์ `templates/` ทั้งหมด 5 หน้าต่างหลัก:

1. **การ์ดคำถามซีพียู (`card_faq_cpu.json`):** แสดงคำอธิบายเปรียบเทียบ Core/Thread และ Clock speed พร้อมรูปไอคอนและปุ่มนำทางไปเรื่องการ์ดจอต่อ
2. **การ์ดคำถามการ์ดจอ (`card_faq_gpu.json`):** อธิบายขนาด VRAM ความละเอียดจอภาพ 2K/4K และข้อแตกต่างระหว่างค่ายสีเขียว (NVIDIA) และค่ายสีแดง (AMD)
3. **การ์ดคำถามหน่วยความจำแรม (`card_faq_ram.json`):** ให้ข้อมูลมาตราส่วนความจุ (8GB, 16GB, 32GB) ตามสัดส่วนของงาน และเปรียบเทียบมาตรฐาน DDR4 กับ DDR5
4. **การ์ดเปรียบเทียบ SSD vs HDD (`card_faq_ssd_hdd.json`):** แสดงภาพเปรียบเทียบความเร็วสายเชื่อมต่อ SATA และชิปความเร็วสูง M.2 NVMe รวมถึงตารางอายุงานการเขียนข้อมูล (TBW)
5. **การ์ดคำแนะนำความเร็วเครื่อง (`card_optimize_performance.json`):** บล็อกข้อความสีสันการจูนระบบ 4 ขั้นตอนพร้อมไอคอนประกอบดูง่ายเด่นชัด

## 4. ฐานข้อมูลที่เกี่ยวข้อง (Database Interactions)

* **ความประหยัดทรัพยากรฐานข้อมูล (Resource Optimization):**
  * เนื่องจากฟังก์ชันนี้เป็นคำสั่งตอบคำถามเชิงให้ข้อมูลการศึกษาทั่วไป (Static Knowledge Inquiry) ระบบจึงไม่ได้ทำการเรียกอ่านคลังสินค้า `hardware_db.json` และไม่มีการบันทึกสถิติลงใน SQLite `analytics.db` เพื่อลดภาระโหลดเขียนฐานข้อมูลและเพิ่มความรวดเร็วในการแสดงผลข้อมูล

## 5. ส่วนต่อประสานโปรแกรมประยุกต์ที่เกี่ยวข้อง (APIs)

* **Rasa Action Server Integration:**
  * **พอร์ตที่ใช้:** HTTP POST `http://127.0.0.1:5055/webhook`
  * **บทบาท:** ส่งคำขอ Intent `ask_*` จากบอท Rasa Core มาประมวลผลสุ่มหาดัชนีและดึง Payload การ์ดความรู้ที่แมปไว้
* **LINE Messaging API:**
  * **Endpoint:** `https://api.line.me/v2/bot/message/push`
  * **บทบาท:** ส่ง (Push) บล็อก Flex Message JSON ที่ผ่านการคัดเลือกสุ่มกลับไปตอบในห้องแชทของ LINE
