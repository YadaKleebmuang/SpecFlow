# S02: กลุ่มเป้าหมาย (Persona) และ Use Case Specification

---

## 1. ผู้มีส่วนได้ส่วนเสียในระบบ (Stakeholders)
1. **ผู้ใช้งานทั่วไป (End Users):** กลุ่มผู้บริโภคที่ต้องการจัดซื้อสเปคคอมพิวเตอร์ประกอบใหม่ หรือผู้ใช้งานที่มีคอมพิวเตอร์อยู่แล้วแต่มีปัญหาเครื่องทำงานช้าและต้องการคำแนะนำในการปรับปรุงอัปเกรดอุปกรณ์
2. **ผู้ดูแลระบบและนักพัฒนา (Administrators / Developers):** ผู้ควบคุมระบบแชทบอท ทำหน้าที่ตรวจสอบความถูกต้องของสถาปัตยกรรมปรับแต่งโมเดลประมวลผลภาษา (NLU) ปรับปรุงคลังข้อมูลราคาและประเภทอะไหล่ไอที (`hardware_db.json`) และการรันสถิติสืบค้นเชิงลึกจาก SQLite เพื่อการประเมินผล
3. **ผู้จำหน่ายฮาร์ดแวร์ภายนอก (Hardware Vendors / Retailers):** กลุ่มธุรกิจร้านค้าอุปกรณ์คอมพิวเตอร์ที่ได้รับผลประโยชน์ทางอ้อมจากการที่ลูกค้านำสเปคแนะนำจากแชทบอทไปใช้ประกอบการสั่งซื้อสินค้าจริงในเชิงพาณิชย์

## 2. กลุ่มเป้าหมายจำลอง (User Persona)

### Persona 1: กลุ่มผู้บริโภคสายเล่นเกม (Gamer Profile)
* **ชื่อ-นามสกุล:** นายศุภชัย ใจดี (เป๊ปซี่)
* **อายุ:** 21 ปี
* **อาชีพ:** นักศึกษาระดับปริญญาตรี คณะวิศวกรรมศาสตร์
* **พฤติกรรมและความคุ้นเคยเทคโนโลยี:** ใช้งานสมาร์ตโฟนและคอมพิวเตอร์ระดับสูง ท่องสื่อสังคมออนไลน์ นิยมใช้โปรแกรมแชท LINE เป็นหลักในการติดต่อสื่อสาร
* **ปัญหาและข้อจำกัด (Pain Points):**
  * มีงบประมาณจำกัดในการประกอบคอมพิวเตอร์ใหม่สำหรับการเล่นเกมระดับสูง
  * มีความรู้ด้านฮาร์ดแวร์พื้นฐานแต่ไม่สามารถคำนวณความเข้ากันได้ เช่น ไม่แน่ใจเรื่องกำลังไฟที่เหมาะสมของ Power Supply (PSU) หรือขนาดเมนบอร์ดที่จะนำไปใส่ในเคสคอมพิวเตอร์
  * กังวลว่าจะจัดสเปคไม่สมดุล เกิดคอขวด หรือซื้อมาประกอบจริงแล้วเปิดระบบไม่ติด
* **ความต้องการและเป้าหมาย (Goals):**
  * ต้องการจัดสเปคคอมพิวเตอร์เล่นเกมที่มีประสิทธิภาพสมดุลสูงสุดในกรอบงบประมาณ 25,000 บาท
  * ต้องการสเปคคอมพิวเตอร์ที่รองรับการอัปเกรดในอนาคต (Future Upgrade) โดยสามารถระบุเงื่อนไขให้บอทบังคับเป็น Socket ใหม่ล่าสุดและแรม DDR5 ได้อย่างแม่นยำ
  * ต้องการคำสั่งคำตอบที่เข้าใจง่ายและสรุปราคารวมอย่างรวดเร็ว

### Persona 2: กลุ่มสายงานความคิดสร้างสรรค์ (Content Creator Profile)
* **ชื่อ-นามสกุล:** นางสาวธนพร พรประเสริฐ (ส้ม)
* **อายุ:** 26 ปี
* **อาชีพ:** นักตัดต่อวิดีโอและกราฟิกดีไซเนอร์อิสระ (Freelance Content Creator)
* **พฤติกรรมและความคุ้นเคยเทคโนโลยี:** ใช้งานคอมพิวเตอร์เป็นงานประจำ มีความคุ้นเคยกับซอฟต์แวร์ตัดต่อ (Premiere Pro, Blender) แต่ไม่ได้ติดตามข่าวสารเกี่ยวกับฮาร์ดแวร์ไอทีรุ่นใหม่
* **ปัญหาและข้อจำกัด (Pain Points):**
  * เครื่องคอมพิวเตอร์เดิมเริ่มมีปัญหาเรนเดอร์งานวิดีโอขนาด 4K ช้ามากจนส่งผลกระทบต่องาน
  * มีงบประมาณอัปเกรดจำกัด ไม่ต้องการลงทุนซื้อคอมพิวเตอร์ใหม่ทั้งชุด
  * ไม่ทราบว่าความล่าช้าเกิดจากอุปกรณ์ชิ้นใด (คอขวดที่ CPU, RAM หรือความจุจานหมุน) และไม่ทราบตัวเลือกอัปเกรดที่คุ้มราคา
* **ความต้องการและเป้าหมาย (Goals):**
  * ต้องการป้อนรายละเอียดสเปคเครื่องคอมพิวเตอร์เดิมเพื่อให้แชทบอทประเมินจุดชำรุดหรือคอขวด (Bottleneck Analysis)
  * ต้องการได้รับข้อเสนอแนวทางการเปลี่ยนอัปเกรดชิ้นส่วนเป็นชิ้นๆ พร้อมการคำนวณราคาประเมินงบประมาณที่จะต้องใช้ทั้งหมด

## 3. เส้นทางการใช้งานระบบของผู้ใช้ (User Journey Map)

เส้นทางการใช้งานของผู้ใช้ทั่วไปผ่านการโต้ตอบกับ LINE Chatbot แบ่งออกเป็น 5 ระยะหลัก ดังนี้:

| ระยะการใช้งาน (Phases) | 1. ค้นพบระบบ (Discovery) | 2. เริ่มสื่อสาร (Initiation) | 3. ป้อนเงื่อนไข (Input Collection) | 4. รับข้อเสนอแนะนำ (Recommendation) | 5. การดำเนินการผล (Execution) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **การกระทำของผู้ใช้ (Actions)** | ค้นหาและแอดไลน์ไอดีของแชทบอทบ๊อบบี้ "SpecFlow" เพื่อเริ่มต้นการใช้งาน | พิมพ์คำว่า "จัดสเปคคอม" หรือ "อยากอัปเกรดคอม" ลงในกล่องแชท | เลือกปุ่มความต้องการใช้งาน (งานออฟฟิศ/เกม/กราฟิก) ระบุจำนวนงบประมาณ และยืนยันความต้องการอัปเกรดในอนาคต | ได้รับการ์ดสรุปสเปคแนะนำในรูปแบบ LINE Flex Message สวยงาม | นำสเปคแนะนำแยกชิ้นพร้อมราคาประเมิน ไปสั่งซื้อหน้าร้านไอทีภายนอก |
| **สิ่งที่ระบบดำเนินการ (System Process)** | ระบบตอบรับอัตโนมัติเปิดหน้าต่างคุยแรกรวมถึงส่งการ์ดต้อนรับ (Greet Card) | ประมวลคำภาษาไทย (ลบคำสร้อย/แก้คำผิด) ระบุ Intent เข้าสู่ระบบแบบฟอร์ม (Form Loop) | ส่งตัวเลือกแบบด่วน (Quick Reply) และบันทึกค่าลงในช่อง Slot | เรียก SpecRecommender/UpgradeAdvisor ประมวลความเข้ากันได้ ดึงข้อมูล JSON และจัดประวัติลง SQLite | สรุปรายงานสถิติใช้งานใน SQLite ในภาพรวมของสเปคที่ลูกค้าทำรายการ |
| **ความรู้สึกผู้ใช้งาน (User Experience)** | สะดวกสบาย เข้าถึงง่ายผ่านแอปพลิเคชันที่มีอยู่ในสมาร์ตโฟน | รู้สึกได้รับการบริการที่รวดเร็วทันใจ มีปุ่มโต้ตอบที่ชัดเจน | ประหยัดเวลา ไม่ต้องพิมพ์ข้อความขนาดยาวด้วยตนเอง | ประทับใจใน UI การ์ดแสดงผล แยกรายละเอียดชัดเจนและราคาแม่นยำ | มั่นใจในการนำข้อมูลไปซื้อประกอบจริงเนื่องจากมีกฎความเข้ากันได้ควบคุม |

## 5. คำอธิบายรายละเอียดการใช้งานยูสเคส (Use Case Descriptions)

ต่อไปนี้เป็นเอกสารแสดงรายละเอียดโฟลว์การทำงานของ ยูสเคสหลักที่สำคัญที่สุด 2 รายการ ได้แก่ **UC01** และ **UC02**

### 5.1. คำอธิบายรายละเอียด Use Case UC01: จัดสเปคคอมพิวเตอร์ตามความต้องการ (Build PC Specification)

* **รหัสยูสเคส:** UC01
* **ชื่อยูสเคส:** จัดสเปคคอมพิวเตอร์ตามความต้องการ (Build PC Specification)
* **ผู้ดำเนินการหลัก (Primary Actor):** ผู้ใช้งานทั่วไป (End User)
* **เงื่อนไขก่อนเริ่มทำงาน (Preconditions):**
  1. ผู้ใช้ได้ทำการติดตั้งโปรแกรมแชท LINE และเพิ่มแชทบอท SpecFlow เป็นเพื่อนเรียบร้อยแล้ว
  2. เซิร์ฟเวอร์หลัก Rasa, Action Server และอินเทอร์เน็ตเกตเวย์ Ngrok รันทำงานอย่างเป็นปกติ
* **เงื่อนไขหลังสิ้นสุดการทำงาน (Postconditions):**
  1. ระบบจัดสเปคประกอบเสร็จสิ้นจำนวน 8 อุปกรณ์ตามข้อจำกัดความเข้ากันได้ทางกายภาพ
  2. ราคารวมของสเปคที่ประกอบไม่เกินขีดจำกัดงบประมาณของผู้ใช้ (ยกเว้นระบบคัดชิ้นส่วนถูกสุดแล้วยังเกินงบ จะแจ้งเตือน)
  3. บันทึกข้อมูลประวัติการจัดสเปคของลูกค้ารายนี้ลงในตาราง `user_searches` บน SQLite `analytics.db` สำเร็จ
  4. ผลลัพธ์สเปคและราคาถูกจัดแสดงในรูปแบบ LINE Flex Message บนจอภาพของผู้ใช้งาน

#### ขั้นตอนการทำงานปกติ (Main Flow)

| ลำดับที่ | การกระทำของผู้ใช้ (Actor Actions) | การตอบสนองของระบบ (System Responses) |
| :--- | :--- | :--- |
| 1 | ผู้ใช้พิมพ์ความต้องการจัดสเปค เช่น "จัดสเปคคอม" หรือส่งข้อความที่มีงบ/การใช้งานมาด้วย | ระบบทำความสะอาดข้อความด้วย Preprocessor และจำแนกเจตนาเป็น `build_pc` จากนั้นเปิดการทำงานของแบบฟอร์ม `build_pc_form` |
| 2 | - | ระบบตรวจสอบข้อมูล Slot: หากยังขาดข้อมูลส่วนใด (งบประมาณ, ประเภทการใช้งาน, ความต้องการเผื่ออนาคต) ระบบจะส่งข้อความถามพร้อมแสดงปุ่มตอบด่วน (Quick Reply) |
| 3 | ผู้ใช้กดเลือกหรือระบุงบประมาณ (เช่น 25000) และประเภทงาน (เช่น เล่นเกม) | ระบบบันทึกข้อมูลเข้าช่อง Slot และนำข้อมูลไปแปลงค่าเงินพร้อมคัดแยกรูปแบบการใช้งาน |
| 4 | ผู้ใช้กดยืนยันความต้องการเผื่ออัปเกรดในอนาคต (ใช่/ไม่ใช่) | ระบบบันทึก Slot `future_upgrade` ครบสมบูรณ์ จากนั้นส่งข้อมูลทั้งหมดไปประมวลผลที่ Action Server (Port 5055) |
| 5 | - | ระบบเรียกใช้ `SpecRecommender` ดำเนินการ 2 ขั้นตอน:<br>1. จัดสรรงบตามสัดส่วนการใช้งาน (Core & Secondary)<br>2. คัดสรรอุปกรณ์และตรวจสอบความเข้ากันได้ (Socket, RAM type, PSU Wattage, Case, Cooler) |
| 6 | - | ระบบทำการบันทึกประวัติธุรกรรมสืบค้นลง SQLite database `analytics.db` |
| 7 | - | ระบบเรียกใช้งาน `flex.py` ประกอบ Flex Message การ์ดสเปคคอมแนะนำ และส่งผลลัพธ์ผ่าน LINE API กลับไปให้ผู้ใช้ |

#### ขั้นตอนการทำงานที่ต่างออกไป (Alternative Flows)

* **Alternative Flow A: ผู้ใช้งานระบุงบประมาณต่ำกว่าขอบเขตขั้นต่ำที่กำหนด:**
  * ในขั้นตอนที่ 3 หากระบบแปลงค่าเงินแล้วพบว่างบประมาณมีค่าน้อยกว่า 10,000 บาท
  * ระบบจะยุติฟอร์ม `build_pc_form` ทันที และส่งข้อความชี้แจงผู้ใช้ว่า: *"ขออภัยครับ งบประมาณต่ำเกินไปในการประกอบคอมพิวเตอร์ครบชุด ลองเพิ่มงบเป็นสัก 15,000 - 20,000 บาทขึ้นไปดูไหมครับ?"*
* **Alternative Flow B: คลังอุปกรณ์มีจำกัดทำให้ราคารวมของสเปคจำเป็นเกินงบประมาณ:**
  * ในขั้นตอนที่ 5 หากราคาอุปกรณ์รวมของ 8 ชิ้นจำเป็นมีค่าสูงกว่างบประมาณที่ผู้ใช้ตั้งไว้เกิน 10%
  * ระบบจะจัดสเปคจนจบโดยหยิบชิ้นส่วนที่ถูกที่สุดที่รองรับ และในขั้นตอนที่ 7 ระบบจะเพิ่มกล่องข้อความแจ้งเตือนสีส้ม (Warning Box) ระบุว่า: *"สเปคนี้มีราคาเกินงบไปเล็กน้อย เนื่องจากอุปกรณ์ในฐานข้อมูลมีจำกัดและต้องเลือกตัวที่ใช้งานร่วมกันได้ครับ"*

### 5.2. คำอธิบายรายละเอียด Use Case UC02: วิเคราะห์การอัปเกรดคอมพิวเตอร์เดิม (Analyze PC Upgrade)

* **รหัสยูสเคส:** UC02
* **ชื่อยูสเคส:** วิเคราะห์การอัปเกรดคอมพิวเตอร์เดิม (Analyze PC Upgrade)
* **ผู้ดำเนินการหลัก (Primary Actor):** ผู้ใช้งานทั่วไป (End User)
* **เงื่อนไขก่อนเริ่มทำงาน (Preconditions):**
  1. ผู้ใช้เพิ่มบอท SpecFlow เป็นเพื่อนในแอปพลิเคชัน LINE แล้ว
  2. เซิร์ฟเวอร์และพอร์ตเชื่อมต่อพร้อมรับส่งข้อมูล
* **เงื่อนไขหลังสิ้นสุดการทำงาน (Postconditions):**
  1. ระบบระบุจุดบกพร่องของฮาร์ดแวร์เดิม ตรวจวิเคราะห์และแสดงผลลัพธ์ภาวะคอขวดระบบอย่างเป็นรูปธรรม
  2. แนะนำรายการอะไหล่แนะนำเปลี่ยนอัปเกรดพร้อมสรุปยอดรวมค่าใช้จ่ายลงในการ์ด Flex Message และแสดงแก่ผู้ใช้
  3. บันทึกสถิติบันทึกการอัปเกรดลงในตารางประวัติผู้ใช้งาน SQLite สำเร็จ

#### ขั้นตอนการทำงานปกติ (Main Flow)

| ลำดับที่ | การกระทำของผู้ใช้ (Actor Actions) | การตอบสนองของระบบ (System Responses) |
| :--- | :--- | :--- |
| 1 | ผู้ใช้ส่งข้อความแจ้งความต้องการ เช่น "อยากอัปเกรดคอมพิวเตอร์เดิม" หรือพิมพ์บอกสเปคเก่าเข้ามา | ระบบประมวลข้อความ ตรวจจับเจตนาเป็น `upgrade_pc` และเรียกใช้ฟอร์ม `upgrade_pc_form` เพื่อเก็บข้อมูล |
| 2 | - | ระบบส่งข้อความถามรายละเอียดสเปคปัจจุบันและประเภทการใช้งานที่ต้องการนำไปใช้หลังอัปเกรด |
| 3 | ผู้ใช้ป้อนรายละเอียดสเปคเดิม (เช่น "i5 4570, ram 8gb, hdd 1tb, gtx 1050ti") และระบุงานที่จะใช้ (เช่น "ตัดต่อวิดีโอ") | ระบบนำข้อความไปลบขีดกลางและช่องว่างเพื่อแปลงรูปแบบคำ ป้อนค่าลง Slot `current_specs` และ `usage` |
| 4 | - | ฟอร์มทำงานเสร็จสมบูรณ์ ส่งคำสั่งเข้าประมวลผลที่โมดูล `UpgradeAdvisor` ใน Action Server |
| 5 | - | ระบบดำเนินตรรกะวิเคราะห์สเปคเดิม:<br>1. เช็คความจุแรมต่ำ (4GB/8GB) $\rightarrow$ แนะนำเพิ่มแรม<br>2. เช็ค HDD จานหมุน $\rightarrow$ แนะนำเปลี่ยนใช้ SSD<br>3. เช็ค GPU เก่า $\rightarrow$ แนะนำการ์ดจอรุ่นใหม่<br>4. เช็ค CPU เก่า/เจนเนอเรชันเก่า $\rightarrow$ แนะนำอัปเกรด CPU พร้อมบอร์ดแรม<br>5. วิเคราะห์ภาวะคอขวด CPU/GPU Bottleneck เชิงลึก |
| 6 | - | ระบบลงบันทึกข้อมูลการสืบค้นลง SQLite database `analytics.db` โดยคำนวณราคายอดแนะนำอัปเกรดสะสม |
| 7 | - | ระบบสร้าง Flex Message การ์ดรายการอัปเกรดแนะนำ และส่งกลับไปแสดงผลบนห้องแชท LINE ของผู้ใช้ |

#### ขั้นตอนการทำงานที่ต่างออกไป (Alternative Flows)

* **Alternative Flow A: ผู้ใช้งานป้อนสเปคสั้นเกินไปหรือระบบไม่พบคำสำคัญอุปกรณ์เก่า:**
  * ในขั้นตอนที่ 3 หากผู้ใช้ส่งเฉพาะข้อความสั้นๆ ที่ระบบไม่สามารถระบุฮาร์ดแวร์เดิมหรือคีย์เวิร์ดที่เป็นตัวกรองได้เลย
  * ระบบจะแสดงผลข้อความแนะนำทั่วไป เช่น: *"จากสเปคที่คุณแจ้งมา ถือว่ายังใช้งานได้ดีครับ แต่ถ้าต้องการอัปเกรดเพื่อรองรับการใช้งานด้าน {usage} ผมแนะนำให้ลองเพิ่ม RAM เป็น 32GB หรือเปลี่ยนการ์ดจอรุ่นใหม่ขึ้นครับ"* โดยราคารวมแนะนำเริ่มต้นกำหนดเป็น 0 บาท
