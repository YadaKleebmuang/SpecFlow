# S09: ระบบการออกแบบส่วนติดต่อผู้ใช้และประสบการณ์ผู้ใช้งาน (UI Design System and UX Specification)

---

## 1. ระบบสีในการออกแบบ (Color Palette)

เนื่องจากระบบแสดงผลแอปพลิเคชัน SpecFlow อยู่ภายใต้ข้อจำกัดของเฟรมเวิร์ก LINE Flex Message ระบบสีจึงถูกกำหนดไว้เป็นรหัสสี Hex Color เพื่อสร้างความน่าเชื่อถือ สแกนอ่านข้อมูลได้รวดเร็ว และมีความเป็นกลางในฐานะบอทให้คำปรึกษาไอที:

| บทบาทการใช้งาน (Color Role) | รหัสสี (Hex Code) | ตัวอย่างการใช้งานจริงในระบบ (Usage Context) |
| :--- | :--- | :--- |
| **Primary Theme (Teal/Turquoise)** | `#1ABC9C` | ใช้เน้นหัวข้อหลัก ชื่อรุ่นสเปคที่โดดเด่น และปุ่มตกลงหลัก |
| **Secondary Dark (Navy/Slate)** | `#2C3E50` | สีสำหรับตัวอักษรชื่อเรื่องหลัก และสีพื้นหลังเน้นความมั่นคง |
| **Text Primary (Dark Slate)** | `#34495E` | สีตัวอักษรเนื้อความทั่วไป เช่น รายละเอียดรุ่นชิ้นส่วน |
| **Text Secondary (Medium Gray)** | `#7F8C8D` | สีของป้ายกำกับด้านหน้าชิ้นส่วน เช่น CPU, Mainboard |
| **Warning/Alert Color (Orange)** | `#D35400` | ใช้กับการ์ดเตือนงบประมาณบานปลาย และคำนำหน้าไอคอนเตือน `⚠️` |
| **Disclaimer Gray (Light Gray)** | `#95A5A6` | ใช้กับตัวอักษรข้อความหมายเหตุและปฏิเสธความรับผิดชอบ |
| **Divider Line (Light border)** | `#E5E5E5` | สีของเส้นคั่นบอกขอบเขตระหว่างบล็อกอุปกรณ์ (`separator`) |

## 2. อักษรศาสตร์และการจัดวาง (Typography)

เนื่องจากแชทบอททำงานบนแอปพลิเคชัน LINE หน้าบ้าน แบบตัวอักษร (Font Family) จะขึ้นตรงกับระบบปฏิบัติการของอุปกรณ์เครื่องปลายทางของผู้ใช้ (เช่น `San Francisco` ใน iOS หรือ `Roboto` ใน Android) 

อย่างไรก็ตาม ระบบได้กำหนดมาตราส่วนขนาดอักษร (Type Scale Matrix) ผ่านค่าพารามิเตอร์ของ Flex Message เพื่อความสวยงามและมีระดับขั้นตอน (Visual Hierarchy) ในการอ่าน ดังตารางนี้:

| ขนาดคีย์ (Scale Key) | ขนาดเทียบเคียงพิกเซล | ความหนาอักษร (Weight) | การนำไปใช้ในการ์ด (Application) |
| :--- | :--- | :--- | :--- |
| **`xxs`** | ~9 px | Regular | ข้อความ disclaimer หมายเหตุ และข้อจำกัดเชิงกฎหมาย |
| **`xs`** | ~12 px | Regular | รายการข้อแนะนำการอัปเกรดแบบ Bullet point |
| **`sm`** | ~14 px | Regular / Bold | รายละเอียดชื่อสินค้า และราคาส่วนประกอบย่อย |
| **`md`** | ~16 px | Regular | ป้ายชื่อประเภท เช่น CPU, Mainboard, RAM ในการ์ดสเปค |
| **`lg`** | ~19 px | Bold | หัวข้อหลักของการ์ด เช่น "สเปคคอมแนะนำสำหรับเล่นเกม" |
| **`xl`** | ~22 px | Bold | ยอดราคารวมสุทธิเด่นชัดเพื่อการตัดสินใจของผู้ใช้ |

## 3. สัญลักษณ์ไอคอน (Iconography System)

แชทบอทเลือกใช้สัญลักษณ์ **Emoji และ Unicode Symbols** มาตรฐานเพื่อทำหน้าที่เป็นไอคอนแทนไฟล์ภาพ เพื่อความรวดเร็วในการโหลดแสดงผลและความเข้ากันได้ข้ามแพลตฟอร์ม (Cross-platform compatibility) โดยมีข้อกำหนดความหมายดังนี้:

* **`🖥️` (Monitor):** สัญลักษณ์แทนการจัดสเปคคอมพิวเตอร์ประกอบชุดใหม่ (Smart PC Builder)
* **`📈` (Chart Up):** สัญลักษณ์แทนการขอคำปรึกษาอัปเกรดอุปกรณ์เดิม (Upgrade Advisor)
* **`💡` (Bulb):** สัญลักษณ์แทนหัวข้อคำแนะนำทางเทคนิคการปรับตั้งค่าระบบ (Performance Optimization)
* **`💬` (Chat):** สัญลักษณ์ใช้ทักทายหรือการเริ่มพิมพ์ข้อความ
* **`❓` / `[?]` (Question):** สัญลักษณ์ระบุประโยคคำถามหรือสเตตัสที่รอค่าจากผู้ใช้งาน
* **`⚠️` (Warning):** ไอคอนเตือนงบประมาณเกิน หรือสัญลักษณ์คำเตือนข้อสงวนสิทธิ์ Disclaimer

## 4. ส่วนประกอบอินเทอร์เฟซผู้ใช้ (UI Components Design)

โครงสร้างหน้ากาก Flex Message แบ่งกล่องออกเป็นส่วนประกอบสำเร็จรูป (Reusable UI Components):

1. **Header Block (กล่องหัวเรื่อง):** สีพื้นหลังเทาอ่อนหรือขาว ข้อความกึ่งกลาง ขนาดตัวหนา `lg` บอกวัตถุประสงค์
2. **Hero Section (รูปภาพแสดงผล):** ใช้ภาพแบนเนอร์คอมพิวเตอร์ประกอบขนาดสากล เพื่อกระตุ้นความน่าดึงดูด
3. **Body Component (แผงตารางชิ้นส่วน):** 
   * จัดการวางแผนแบบโครงสร้าง Flexbox แนวนอน (Horizontal Layout)
   * ฝั่งซ้าย (Flex = 2): แสดงหมวดหมู่ชิ้นส่วนตัวอักษรสีเทา `#aaaaaa`
   * ฝั่งขวา (Flex = 5): แสดงชื่อรุ่นฮาร์ดแวร์และราคาจริงในวงเล็บ ตัวอักษรสีเข้ม `#666666` เพื่อให้อ่านง่าย
4. **Separator Line (เส้นคั่นส่วน):** เส้นบางแนวนอนเพื่อจัดหมวดหมู่ข้อมูลไม่ให้รกสายตา
5. **Warning Container (กล่องคำเตือน):** กล่องข้อความสีส้มอิฐขนาดย่อพิเศษ `xxs` แสดงแจ้งเตือนงบประมาณบานปลาย
6. **Quick Reply Buttons (ปุ่มตอบกลับด่วน):** ปุ่มวงรีแนบมากับแชทด้านล่าง สำหรับคลิกข้อมูลโดยไม่ต้องพิมพ์

## 5. หลักการออกแบบประสบการณ์ผู้ใช้ (UX Principles)

ระบบประยุกต์ใช้หลักการออกแบบตามมาตรฐาน **User Experience (UX) Principles** เพื่อความลื่นไหลและพึงพอใจสูงสุด:

### 5.1. หลักการลดภาระการพิมพ์ของผู้ใช้ (Minimize Interaction Cost)
* ผู้ใช้ไม่จำเป็นต้องพิมพ์ประโยคคำตอบขนาดยาว บอทจะใช้ Quick Reply เสนอปุ่มตัวเลือกให้กดในทันที เช่น การกดเลือกปุ่ม "เล่นเกม", "ทำงานกราฟิก" ในขั้นตอนเลือกประเภทงาน หรือปุ่ม "ใช่", "ไม่" ในส่วนของการเผื่ออัปเกรดในอนาคต

### 5.2. หลักการความโปร่งใสและสร้างความเชื่อมั่น (Transparency & Trust)
* การจัดสเปคคอมพิวเตอร์จะเปิดเผยราคาแยกส่วนประกอบของอุปกรณ์ทุกชิ้นอย่างชัดเจน พร้อมแสดงยอดราคารวมจริง ซึ่งทำให้ผู้ใช้ตรวจสอบราคาและนำไปเปรียบเทียบกับท้องตลาดจริงได้ ช่วยสร้างความโปร่งใสและน่าเชื่อถือให้แก่ระบบบอท

### 5.3. หลักการป้องกันความผิดพลาดล่วงหน้า (Error Prevention)
* เมื่อระบบวิเคราะห์การอัปเกรด (Upgrade Advisor) หากผู้ใช้ป้อนข้อความสเปคเดิมเข้ามา แต่ระบบประเมินแล้วไม่พบคีย์เวิร์ดของชิ้นส่วนเก่า บอทจะทำการส่งหน้าจอตอบกลับคำแนะนำทางเลือกทั่วไป (Fallback Recommendation) ทันทีเพื่อไม่ให้ระบบค้างหรือเกิดเงียบหาย (Silent Failure)

## 6. การตอบสนองตามขนาดหน้าจอและระบบนำทาง (Responsive & Navigation)

* **ความยืดหยุ่นในการแสดงผล (Responsive Design):** 
  * หน้าจอ Flex Message ใช้หลักการจัดหน้าแบบ **CSS Flexbox** ข้อมูลความกว้างจะยืดและหดตัวโดยอัตโนมัติ (Flexible Width Auto-wrapping) 
  * ข้อความจะตัดคำขึ้นบรรทัดใหม่ด้วยพารามิเตอร์ `"wrap": true` เสมอ เพื่อรับประกันว่ารายละเอียดชื่อรุ่นฮาร์ดแวร์ขนาดยาวจะไม่ล้นทะลุออกไปนอกกรอบหน้าต่างแชทของ LINE ในสมาร์ตโฟนหน้าจอขนาดเล็ก
* **ระบบการนำทางที่ชัดเจน (Clear Navigation Flow):**
  * ทุกการ์ดผลลัพธ์ (เช่น การ์ดแนะนำสเปค หรือการ์ดความรู้ FAQ) จะมีปุ่มสั่งการด้านล่างสำหรับการโอนย้ายสถานะเสมอ เช่น ปุ่มกดเพื่อสอบถามชิ้นส่วนอื่นต่อ เพื่ออำนวยความสะดวกในการใช้งานแบบต่อเนื่อง
