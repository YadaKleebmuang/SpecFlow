## 1. ขั้นตอนการทำงาน (Workflow)

ฟังก์ชันแนะนำการอัปเกรดคอมพิวเตอร์เดิม (Upgrade Advisor) ทำการรวบรวมรายละเอียดคอมพิวเตอร์ปัจจุบันของผู้ใช้เพื่อตรวจเช็คคอขวดและให้คำแนะนำเปลี่ยนชิ้นส่วนตามขั้นตอนโฟลว์ดังนี้:

```text
[ผู้ใช้] พิมพ์ "อยากอัปเกรดคอม" -> [Rasa NLU] ระบุเจตนา upgrade_pc
  -> [Rasa Core] เปิดฟอร์ม upgrade_pc_form
  -> [Rasa Core] เรียก Action ถามสเปคเดิม (Slot: current_specs)
  -> [ผู้ใช้] ระบุคำบรรยายสเปคเดิม (เช่น "ใช้ i3 เจน 4 แรม 8 กิ๊ก hdd 1tb การ์ดจอ gtx 750")
  -> [Rasa Core] เรียก Action ถามการนำไปใช้ (Slot: usage) (แสดงปุ่มตอบด่วน Quick Reply)
  -> [ผู้ใช้] เลือกประเภทงาน (เช่น เล่นเกมหนักๆ, ทำงานทั่วไป)
  -> [Rasa Core] ปิดฟอร์ม upgrade_pc_form และเรียก Action Server พอร์ต 5055: action_recommend_upgrade
  -> [Action Server] เรียก UpgradeAdvisor วิเคราะห์ตัวอักษรและเปรียบเทียบอุปกรณ์
  -> [Action Server] บันทึกสถิติประวัติจัดแนะนำการอัปเกรดลง SQLite (analytics.db)
  -> [Action Server] ประกอบการ์ด LINE Flex Message และส่งคืนผลลัพธ์ผ่าน LINE API
  -> [Rasa Core] สั่งคืนค่า Reset Slots ทุกตัวเป็น Null
```

## 2. ตรรกะการประมวลผลทางธุรกิจ (Business Logic)

ตรรกะการคัดวิเคราะห์และเสนอแนะชิ้นส่วนอัปเกรด ควบคุมผ่านคลาส `UpgradeAdvisor` ในไฟล์ [upgrade_advisor.py](file:///c:/Users/thirs/Downloads/SpecFlow/app/services/recommendation/upgrade_advisor.py) โดยแบ่งเป็นขั้นตอนหลักดังนี้:

### 2.1. การปรับระเบียบข้อความอินพุต (Input Normalization)
เนื่องจากผู้ใช้งานระบุรายละเอียดสเปคมาได้หลายรูปแบบ ระบบจะทำการล้างอักขระพิเศษเพื่อป้องกันความคลาดเคลื่อนในการจับคู่คำ:
1. แปลงตัวอักษรภาษาอังกฤษทั้งหมดเป็นตัวพิมพ์เล็ก
2. ลบเว้นวรรค (Whitespaces) และเครื่องหมายขีดกลาง (`-`) ออกทั้งหมด เช่น:
   * `"Core i5-4570"` $\rightarrow$ `"corei54570"`
   * `"GTX 1050 Ti"` $\rightarrow$ `"gtx1050ti"`
   * `"RAM 8 GB"` $\rightarrow$ `"ram8gb"`

---

### 2.2. ตรรกะการประเมินการอัปเกรดอุปกรณ์ทีละประเภท (Rule-based Analysis)

1. **การตรวจสอบหน่วยความจำ (RAM Checking):**
   * *เงื่อนไข:* หากพบคำว่า `"4gb"`, `"8gb"`, `"ram4"`, หรือ `"ram8"` ในข้อความสเปคเดิม
   * *ผลลัพธ์:* แนะนำให้เพิ่มความจุ RAM โดยคัดเลือกตัวราคาที่ดีที่สุดในคลัง (เช่น เพิ่มแรมเป็น 16GB หรือ 32GB)
2. **การตรวจสอบไดรฟ์เก็บข้อมูล (Storage Checking):**
   * *เงื่อนไข:* หากพบคำว่า `"hdd"`, `"harddisk"`, หรือ `"ฮาร์ดดิสก์"`
   * *ผลลัพธ์:* แนะนำให้ติดตั้งไดรฟ์ความเร็วสูงแบบ SSD เพื่อเป็นที่ตั้งระบบวินโดวส์และซอฟต์แวร์หลักช่วยเพิ่มความเร็วในการเปิดเครื่องและทำงาน
3. **การตรวจสอบการ์ดแสดงผลกราฟิก (GPU Checking):**
   * *เงื่อนไข:* หากพบคำสำคัญการ์ดจอรุ่นเริ่มต้นตระกูลเก่า เช่น `"1030"`, `"730"`, `"750"`, `"1050"`, `"1060"`, `"1650"`, `"rx570"`, `"rx580"` หรือในสล็อตความต้องการระบุประเภทการทำงานแบบ "เล่นเกมหนัก" หรือ "ตัดต่อวิดีโอ"
   * *ผลลัพธ์:* แนะนำให้อัปเกรดการ์ดจอแยกเป็นรุ่นใหม่ขึ้นตามราคาที่เหมาะสม
4. **การตรวจสอบหน่วยประมวลผลกลาง (CPU Checking):**
   * *เงื่อนไข:* ใช้ระบบ Regular Expression ตรวจจับตัวเลขรุ่นซีพียู เช่น `i[3579](\d)\d{3}` เพื่อเช็คซีพียูเจนเนอเรชันที่ 9 หรือเก่ากว่า (Gen 2 - Gen 9) หรือคำศัพท์ `"ryzen3"`, `"ryzen51"`, `"ryzen52"`, `"ryzen53"` (Ryzen 1000 - 3000 series)
   * *ผลลัพธ์:* แนะนำให้อัปเกรดซีพียูเป็นรุ่นล่าสุด พร้อมส่งคำแจ้งเตือนว่า *"อาจต้องเปลี่ยนเมนบอร์ดและแรมร่วมด้วยเพื่อความเข้ากันได้"*

---

### 2.3. ตรรกะวิเคราะห์ภาวะคอขวดระบบ (Bottleneck Analysis Logic)
ระบบจะทำการจับคู่ความขัดแย้งของอุปกรณ์ประสิทธิภาพสูงและต่ำในระบบเดิมของผู้ใช้:

* **วิเคราะห์คอขวดซีพียู (CPU Bottleneck):**
  * *เงื่อนไข:* ตรวจพบ CPU รุ่นเก่าในระบบ (`has_old_cpu`) ร่วมกับการใช้งานการ์ดจอแสดงผลระดับสูงแรงสุด (`has_strong_gpu` เช่น 3060, 4060, 4070, 4090, 6600, 7600, 7800)
  * *ผลลัพธ์:* แจ้งเตือนว่า *"ซีพียูประมวลผลข้อมูลไม่ทันการทำงานของการ์ดจอ (การ์ดจอทำงานได้ไม่เต็ม 100%) แนะนำให้อัปเกรด CPU เป็นอันดับแรกเพื่อดึงพลังการ์ดจอออกมาได้เต็มที่ครับ"*
* **วิเคราะห์คอขวดการ์ดจอ (GPU Bottleneck):**
  * *เงื่อนไข:* ตรวจพบ CPU รุ่นใหม่ที่มีความเร็วเพียงพอแล้ว (`has_modern_cpu`) แต่การ์ดจอแยกอยู่ในเกณฑ์เริ่มต้นรุ่นเก่า (`has_weak_gpu` เช่น 1030, 730, 750, 1050)
  * *ผลลัพธ์:* แจ้งเตือนว่า *"มี CPU รุ่นใหม่ที่แรงเพียงพอแล้ว แต่การ์ดจออยู่ในระดับเริ่มต้น ทำให้การเล่นเกมหรือทำงานกราฟิกติดขัดที่การ์ดจอเป็นหลัก แนะนำอัปเกรดการ์ดจอเป็นรุ่นใหม่กว่านี้ครับ"*
* **วิเคราะห์การ์ดจอระดับเริ่มต้นมาก (Entry-level GPU warning):**
  * *เงื่อนไข:* ตรวจพบ GPU ระดับพิมพ์งานทั่วไป เช่น GT 1030 หรือ GT 730
  * *ผลลัพธ์:* แจ้งเตือนว่า *"ไม่เหมาะสำหรับการเล่นเกม 3D ยุคใหม่หรือการทำงานตัดต่อกราฟิก แนะนำอัปเกรดเป็นการ์ดจอแยกที่แรงขึ้นค่ะ"*

## 3. หน้าจอและอินเทอร์เฟซโต้ตอบ (UI Screenshots & Layouts)

ฟังก์ชันอัปเกรดคอมพิวเตอร์ประยุกต์ใช้งาน Flex Message Templates ในไดเรกทอรี `templates/`:

1. **หน้าจอสอบถามรายละเอียดสเปคเดิม (`card_ask_current_specs.json`):**
   * การ์ดคำถามสไตล์สีเข้ม แสดงคำแนะนำตัวอย่างการป้อนสเปคคร่าวๆ เพื่อชี้แนวทางให้ผู้ใช้กรอกข้อความ
2. **หน้าจอสเก็ตช์คำแนะนำและผลวิเคราะห์อัปเกรด (`card_upgrade_advisor.json`):**
   * **รายการชิ้นส่วนแนะนำ:** แสดงรายการที่ต้องอัปเกรดแยกเป็นข้อๆ (ลบเครื่องหมายความหนา `**` ออกเพื่อความเป็นระเบียบบนมือถือ)
   * **ช่องสรุปงบอัปเกรดสะสม:** แสดงยอดเงินบาทรวมตัวหนาสีน้ำเงิน
   * **กล่องคอขวดระบบ (Bottleneck Warning Box):** แสดงข้อคิดเห็นแจ้งเตือนทางเทคนิคในการแก้ไขคอขวดในระบบ ซึ่งเป็นจุดเด่นหลักของฟังก์ชันนี้

## 5. ส่วนต่อประสานโปรแกรมประยุกต์ที่เกี่ยวข้อง (APIs)

  * **พอร์ตที่ใช้:** HTTP POST `http://127.0.0.1:5055/webhook`
