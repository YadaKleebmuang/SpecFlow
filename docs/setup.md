# 🛠️ คู่มือการติดตั้งและเปิดใช้งานระบบ SpecFlow Chatbot 🚀

คู่มือนี้สรุปขั้นตอนทั้งหมดตั้งแต่การติดตั้งสภาพแวดล้อมจำลอง (venv) การติดตั้งไลบรารีที่จำเป็น การเทรนโมเดล ตลอดจนวิธีการเปิดใช้งานระบบและการประเมินประสิทธิภาพแชทบอท

---

## ⚙️ ขั้นตอนที่ 1: การเตรียมสภาพแวดล้อมจำลอง (Environment Setup)

ในการใช้งานระบบ SpecFlow แนะนำให้ใช้ **Python 3.10** ซึ่งเป็นเวอร์ชันที่มีความเสถียรสูงสุดสำหรับ Rasa 3.x

### 1. สร้าง Virtual Environment (venv)
เปิดหน้าต่าง Terminal (Command Prompt / PowerShell / Git Bash) ขึ้นมาที่โฟลเดอร์หลักของโปรเจกต์ (เช่น `C:\Users\yadak\Desktop\SpecFlow`) แล้วรันคำสั่ง:
```bash
py -3.10 -m venv venv
```
> [!NOTE]
> คำสั่งนี้จะสร้างโฟลเดอร์ชื่อ `venv` ขึ้นมาภายในโฟลเดอร์โปรเจกต์ของคุณ

### 2. เปิดใช้งาน Virtual Environment (Activate venv)
เปิดใช้งานตามประเภท Terminal ที่คุณเลือกใช้:

* **Command Prompt (CMD):**
  ```cmd
  venv\Scripts\activate.bat
  ```
* **Windows PowerShell:**
  ```powershell
  .\venv\Scripts\Activate.ps1
  ```
* **Git Bash:**
  ```bash
  source venv/Scripts/activate
  ```

> [!IMPORTANT]
> หากเปิดใช้งานสำเร็จ จะมีสัญลักษณ์ **`(venv)`** ปรากฏอยู่ที่ต้นบรรทัดคำสั่ง (Command Line Prompt) เสมอ

### 3. ตรวจสอบเวอร์ชัน Python
ตรวจสอบให้มั่นใจว่าระบบจำลองกำลังเรียกใช้เวอร์ชันที่ถูกต้อง:
```bash
python --version
```
*(ผลลัพธ์ที่ถูกต้องควรได้: `Python 3.10.x`)*

## 📦 ขั้นตอนที่ 2: การติดตั้ง Dependencies และไลบรารี

เมื่ออยู่ในสถานะ `(venv)` แล้ว ให้ติดตั้งโมดูลที่จำเป็นต่อการพัฒนาและวิเคราะห์ภาษา:

```bash
# 1. ติดตั้งไลบรารีวิเคราะห์ภาษาไทย (PyThaiNLP)
pip install -r requirements/ai.txt

# 2. ติดตั้ง Rasa, Rasa SDK และ LINE SDK
pip install -r requirements/rasa.txt
```

> [!WARNING]
> ห้ามกดยกเลิกการทำงานกลางคัน (เช่น กด Ctrl+C) จนกว่าการติดตั้งจะเสร็จสมบูรณ์ (`Successfully installed...`) เนื่องจากตัวโปรแกรมมีขนาดใหญ่และอาจใช้เวลาติดตั้ง 3-5 นาที

## 🧠 ขั้นตอนที่ 3: การเทรนโมเดลแชทบอท (Model Training)

ก่อนเริ่มใช้งาน จำเป็นต้องเทรนบอทเพื่อให้เข้าใจตรรกะการตอบกลับภาษาไทย:

```bash
# 1. เข้าไปที่โฟลเดอร์ rasa
cd app/rasa

# 2. สั่งเทรนโมเดล (ไฟล์โมเดลจะถูกสร้างอัตโนมัติในโฟลเดอร์ models/)
rasa train

# 3. ย้ายกลับออกมายังโฟลเดอร์หลักของโปรเจกต์
cd ../..
```

## 🚀 ขั้นตอนที่ 4: การรันเซิร์ฟเวอร์เพื่อเปิดใช้งานระบบ (Running the Bot)

การเปิดใช้งานแชทบอทมี 2 วิธีหลักๆ เลือกใช้วิธีที่สะดวกตามการใช้งานของคุณ:

### 💡 วิธีที่ 1: รันผ่านสคริปต์อัตโนมัติ (แนะนำสำหรับผู้ใช้ทั่วไป)
คุณสามารถรันระบบเซิร์ฟเวอร์ทั้งหมด (Rasa Server, Action Server, และ Ngrok) พร้อมกันในหน้าต่างเดียวด้วยสคริปต์หลัก:
```bash
python run.py
```
> [!TIP]
> หากต้องการปิดหรือหยุดการทำงานของระบบทั้งหมด ให้กดปุ่ม **`Ctrl + C`** ในหน้าต่างสั่งการนี้ได้ทันที

---

### 🛠️ วิธีที่ 2: รันแยกทีละบริการ (สำหรับนักพัฒนา / Debugging)
เปิด Terminal แยกกัน **3 หน้าต่าง (ทุกหน้าต่างต้อง Activate venv และเข้าโฟลเดอร์ให้ถูกต้อง)**

#### 🟢 หน้าต่างที่ 1: เปิด Action Server (ระบบประมวลผลจัดสเปค)
รันเพื่อใช้งาน Custom Action สำหรับวิเคราะห์ชุดคำสั่งจัดสเปคคอมพิวเตอร์:
```bash
cd app/rasa
rasa run actions
```
*(เมื่อพร้อมใช้งานจะขึ้น: `Action endpoint is up and running on http://localhost:5055`)*

#### 🔵 หน้าต่างที่ 2: เปิด Ngrok (เชื่อมต่อไปยัง LINE Webhook)
ทำหน้าที่ส่งข้อมูลจากแอปพลิเคชัน LINE ลงมายังบอทในเครื่องของคุณ:
```bash
ngrok http 5005
```
1. มองหาบรรทัดที่ระบุคำว่า **Forwarding** แล้วคัดลอก URL ที่ขึ้นต้นด้วย `https://` (เช่น `https://abcd-1234.ngrok-free.app`)
2. เข้าสู่ [LINE Developers Console](https://developers.line.biz/) -> เลือกแชทบอทของคุณ -> ไปที่แท็บ **Messaging API**
3. แก้ไขข้อมูลช่อง **Webhook URL** โดยนำลิงก์ Ngrok ที่คัดลอกมาวาง แล้วตามด้วย `/webhooks/line/webhook`
   *ตัวอย่าง:* `https://abcd-1234.ngrok-free.app/webhooks/line/webhook`
4. กดปุ่ม **Verify** (ผลลัพธ์ต้องขึ้นเป็น *Success*) และเปิดใช้งานตัวสวิตช์ **Use webhook** 🟢

#### 🟣 หน้าต่างที่ 3: เปิด Rasa Server (สมองแชทบอทหลัก)
รัน Rasa Server เพื่อทำความเข้าใจประโยค (NLU) และติดต่อสื่อสารกับ LINE:
```bash
cd app/rasa
rasa run --enable-api --cors "*"
```

## 🧪 ขั้นตอนที่ 5: การประเมินผลระบบ (NLU Evaluation)

หากต้องการนำค่าสถิติวัดประสิทธิภาพ (Accuracy, Precision, Recall, F1-Score) ไปใช้งานหรือประกอบรายงานบทที่ 4 สามารถรันคำสั่งทดสอบได้ดังนี้:
```bash
cd app/rasa
rasa test nlu
```
> [!NOTE]
> เมื่อรันเรียบร้อย ผลลัพธ์ในรูปของกราฟ Confusion Matrix และรายงานประสิทธิภาพต่างๆ จะถูกบันทึกไว้ในโฟลเดอร์ `app/rasa/results`
